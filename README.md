# 外贸采购比价助手 (Sourcing Agent)

> 智能解析电子白板（Interactive Flat Panel）PDF 规格书，自动提取关键参数，进行多供应商对比分析，并生成预警报告。

---

## 📁 项目结构

```
sourcing-agent/
├── core.py           # 核心业务逻辑层（MVC Model）
│                     #   - PDF 文本提取
│                     #   - LLM 语义分析
│                     #   - 行业规则预警
│                     #   - 模拟数据生成
│                     #   - 数据导出（Excel/CSV）
├── app.py            # 界面展示层（MVC View + Controller）
│                     #   - Streamlit 网页界面
│                     #   - 侧边栏设置面板
│                     #   - 结果展示与下载
├── requirements.txt  # Python 依赖清单
└── README.md         # 本文件
```

## 🚀 快速开始

### 1. 环境要求

- Python 3.9+
- pip（Python 包管理器）

### 2. 安装依赖

```bash
cd sourcing-agent
pip install -r requirements.txt
```

### 3. 启动应用

```bash
streamlit run app.py
```

启动后浏览器会自动打开 `http://localhost:8501`。

### 4. 使用方式

| 场景 | 操作 |
|------|------|
| **演示模式（无需 API Key）** | 直接上传 PDF 文件（或不传），点击「开始分析」，自动使用内置的 3 家模拟供应商数据展示效果 |
| **真实解析（需 API Key）** | 在侧边栏输入 OpenAI API Key（支持自定义 Base URL），上传真实 PDF 规格书，点击分析 |

## 📊 功能特性

### 🔍 智能参数提取

自动从 PDF 规格书中提取以下字段：

| 字段 | 说明 |
|------|------|
| 厂家名称 | 制造商/品牌 |
| 屏幕尺寸 | 如 65"、75"、86" |
| 亮度(cd/m²) | 屏幕亮度值 |
| 触控技术 | 红外/电容/电磁等 |
| 安卓版本 | 内置系统版本 |
| CPU 型号 | 处理器型号 |
| 内存/存储 | RAM+ROM 配置 |
| 是否含 OPS | 是否内置可插拔电脑 |
| 认证信息 | CE/FCC/SASO 等 |

### ⚠️ 行业规则预警

| 预警类型 | 触发条件 | 级别 |
|----------|----------|------|
| 低亮预警 | 亮度 < 400 cd/m² | 🔴 严重 |
| 卡顿风险 | 内存 < 4GB | 🔴 严重 |
| 认证缺失 | 目标市场要求认证但规格书未包含 | 🟠 提醒 |

### 🌍 目标市场认证检测

支持的市场及对应认证要求：

| 市场 | 必须认证 |
|------|----------|
| 中东 | CE, SASO |
| 欧洲 | CE, RoHS |
| 北美 | FCC, UL |
| 中国 | CCC |
| 东南亚 | CE |

### 📥 报表导出

- **Excel (.xlsx)**：完整的对比表格，适合汇报
- **CSV (.csv)**：UTF-8 BOM 编码，Excel 可直接打开

## ⚙️ 配置说明

所有配置均通过 Streamlit 侧边栏完成，无需修改代码：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| API Base URL | OpenAI 兼容 API 地址 | `https://api.openai.com/v1` |
| API Key | API 密钥（留空则使用模拟数据） | 空 |
| 模型名称 | 使用的 LLM 模型 | `gpt-3.5-turbo` |
| Temperature | 生成温度（0=确定, 1=随机） | 0.0 |
| 目标市场 | 用于认证预警检测 | 中东 |

> **兼容的 API 服务**：OpenAI、OpenRouter、Azure OpenAI、本地 Ollama/LocalAI 等所有 OpenAI 兼容接口。

| 厂家 | 尺寸 | 亮度 | 内存 | 认证 |
|------|------|------|------|------|

## 🏗️ 架构设计

本项目严格遵循 MVC 架构模式：

```
┌─────────────────────────────────────────┐
│              app.py (View)               │
│        Streamlit UI / 用户交互           │
└──────────────────┬──────────────────────┘
                   │ 调用
                   ▼
┌─────────────────────────────────────────┐
│             core.py (Model)              │
│  ┌─────────┐ ┌──────────┐ ┌──────────┐ │
│  │PDF 解析│ │AI 交互   │ │预警规则  │ │
│  └─────────┘ └──────────┘ └──────────┘ │
│  ┌─────────┐ ┌──────────┐              │
│  │模拟数据│ │数据导出  │              │
│  └─────────┘ └──────────┘              │
└─────────────────────────────────────────┘
```

- `core.py` **完全不依赖 Streamlit**，可独立测试和复用
- `app.py` 仅负责界面渲染和用户交互，所有业务逻辑委托给 `core.py`

## 🛡️ 容错机制

- **PDF 解析失败**：自动使用默认值填充，不会崩溃
- **AI API 调用失败**：返回默认值，记录错误日志
- **AI 返回非 JSON**：自动清理 Markdown 标记，失败则使用默认值
- **导出失败**：捕获异常并向用户展示友好错误提示
- **无 API Key**：自动切换为模拟数据模式
