# 外贸采购比价助手 (Sourcing Agent) v2.0

> 专为外贸电子白板（Interactive Flat Panel）采购场景设计的智能分析工具。  
> 自动解析 PDF 规格书 → AI 提取关键参数 → 多供应商对比 → 预警报告 → 一键导出。

---

## 📁 项目结构

```
sourcing-agent/
├── core.py           # 核心业务逻辑层（MVC Model）
│                     #   - PDF 文本提取 (pdfplumber)
│                     #   - LLM 语义分析 (OpenAI 兼容 API)
│                     #   - 行业规则预警（亮度/内存/认证）
│                     #   - 数据导出（Excel/CSV）
├── app.py            # 界面展示层（MVC View + Controller）
│                     #   - Streamlit 网页界面
│                     #   - 侧边栏设置面板
│                     #   - 结果展示与下载
├── requirements.txt  # Python 依赖清单
└── README.md         # 本文件
```

## 🚀 快速开始

### 1. 环境要求

- **Python** 3.9+
- **pip**（Python 包管理器）

### 2. 安装依赖

```bash
cd sourcing-agent
pip install -r requirements.txt
```

### 3. 启动应用

```bash
streamlit run app.py
```

启动后浏览器自动打开 `http://localhost:8501`，即可开始使用。

### 4. 开箱即用

应用启动后默认选中 **阿里云 (免费额度)**，API Key 已预置，无需任何配置：

1. 上传 PDF 规格书文件（支持拖拽，可同时上传多份）
2. 在侧边栏选择目标市场（中东/欧洲/北美/中国/东南亚）
3. 点击「🔍 开始分析」
4. 查看对比表格和预警信息，一键导出 Excel/CSV 报表

## 📊 功能特性

### 🔍 智能参数提取

AI 自动从 PDF 规格书中提取以下 9 个关键字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| 厂家名称 | 制造商 / 品牌 | MaxTouch Technology |
| 屏幕尺寸 | 面板尺寸 | 75" |
| 亮度(cd/m²) | 屏幕亮度 | 450 cd/m² |
| 触控技术 | 触控方案 | 红外触控 |
| 安卓版本 | 内置系统 | Android 12 |
| CPU 型号 | 处理器 | RK3588 |
| 内存/存储 | RAM + ROM | 4GB+32GB |
| 是否含 OPS | 可插拔电脑模块 | 是 |
| 认证信息 | 产品认证 | CE, FCC, RoHS |

### ⚠️ 行业规则预警

| 预警类型 | 触发条件 | 级别 | 说明 |
|----------|----------|------|------|
| 低亮预警 | 亮度 < 400 cd/m² | 🔴 严重 | 室内展示效果可能不佳 |
| 卡顿风险 | 内存 < 4GB | 🔴 严重 | 多任务运行时可能出现卡顿 |
| 认证缺失 | 目标市场要求认证未覆盖 | 🟠 提醒 | 建议补充相关认证 |

### 🌍 目标市场认证检测

| 市场 | 必须认证 |
|------|----------|
| 中东 | CE, SASO |
| 欧洲 | CE, RoHS |
| 北美 | FCC, UL |
| 中国 | CCC |
| 东南亚 | CE |
| 不限 | （跳过认证检测） |

### 📥 报表导出

- **Excel (.xlsx)** — 完整对比表格，含预警信息，适合直接汇报
- **CSV (.csv)** — UTF-8 BOM 编码，Excel 可直接打开，方便二次加工

## ⚙️ AI 平台配置

所有配置通过 Streamlit 侧边栏完成，无需修改代码：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| AI 服务商 | 选择使用的 AI 平台 | 阿里云 |
| API Key | 平台密钥（阿里云/百度已预置） | 自动填充 |
| 模型名称 | LLM 模型 | qwen-turbo |
| Temperature | 生成温度（0=确定, 1=随机） | 0.0 |
| 目标市场 | 出口目的地（用于认证检测） | 不限 |

### 支持的 AI 平台

| 平台 | API Key | 默认模型 | 说明 |
|------|---------|----------|------|
| **阿里云 DashScope** | 已预置 | qwen-turbo | 免费额度充足，推荐使用 |
| **百度千帆 Qianfan** | 已预置 | ernie-speed | 免费额度 |
| **自定义配置** | 自行填入 | gpt-3.5-turbo | 兼容 OpenAI / OpenRouter / Ollama 等 |

> **兼容性**：支持所有 OpenAI 兼容 API 接口，包括但不限于 OpenAI、Azure OpenAI、OpenRouter、LocalAI、Ollama 等。

## 🏗️ 架构设计

本项目严格遵循 **MVC 架构模式**，确保代码可维护、可测试、可复用：

```
┌─────────────────────────────────────────┐
│              app.py (View)               │
│        Streamlit UI / 用户交互            │
└──────────────────┬──────────────────────┘
                   │ 调用
                   ▼
┌─────────────────────────────────────────┐
│             core.py (Model)              │
│  ┌─────────┐ ┌──────────┐ ┌──────────┐ │
│  │PDF 解析 │ │AI 交互   │ │预警规则  │ │
│  └─────────┘ └──────────┘ └──────────┘ │
│  ┌──────────┐                           │
│  │数据导出  │                           │
│  └──────────┘                           │
└─────────────────────────────────────────┘
```

- **`core.py`** — 完全不依赖 Streamlit，可独立测试和复用于其他界面框架
- **`app.py`** — 仅负责界面渲染和用户交互，所有业务逻辑委托给 `core.py`

## 🛡️ 容错机制

| 场景 | 处理策略 |
|------|----------|
| PDF 无法解析（图片型 PDF） | 返回默认值，标记"解析失败"，不崩溃 |
| AI API 调用超时/失败 | 返回默认值，记录错误日志 |
| AI 返回非 JSON 内容 | 自动清理 Markdown 标记，重试解析 |
| 字段提取缺失 | 使用预设默认值填充（"未提供"） |
| 导出模块异常 | 捕获并向用户展示友好错误提示 |
